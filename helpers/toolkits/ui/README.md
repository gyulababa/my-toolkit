helpers/toolkits/ui
Purpose

Frontend-agnostic UI toolkits: spec definitions, state models, and runtime helpers used by services and adapters.

Belongs here

UI spec models, serde, and validation (spec)

UI state models, defaults, migrations, and loader integration (state)

Runtime session helpers and command/event structures (runtime)

Does not belong here

Adapter-specific UI toolkit code (my_toolkit/toolkit_adapters/adapters)

Service orchestration or persistence layout decisions (services, helpers/persist)

Public API (flat list)
